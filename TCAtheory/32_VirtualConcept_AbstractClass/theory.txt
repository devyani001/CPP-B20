Polymorphism
-------------
--> Ability to take more than one form is called Polymorphism.
--> Ex : Overloading, Overriding
--> Compile Time Polymorphism(decision making process) 	: Overloading
--> Run Time Polymorphism     						   	: over-riding 

// Overloading
class ABC 
{
	// Overloading doesn't need Inheritance
	public :
	void display(int i)
	{
		#1
	}

	void display(float f)
	{
		#2
	}
};

ABC ob;
ob.display(5);		// fun call is binded to function body at compile time
ob.display(3.4);


Compile Time Polymorphism / Early Binding / static Linking
-----------------------------------------------------------
--> When program has function over-loading, then function call is binded to function body at compile time is called as Compile time Polymorphism.

Run Time Polymorphism / Late Binding / Dynamic Linking(Runtime)
----------------------------------------------------------------
--> When program has function over-riding, then function call is binded to function body at runtime is called as Runtime Polymorphism.


// Overriding
class P
{
	public :
	void display()
	{
		cout << "IND" << endl;
	}
};

class C : public P
{
	public :
	void display()
	{
		cout << "AUS" << endl;
	}
};

C ob;
ob.display();

output :
---------
IND
AUS

ABSTRACT CLASS
---------------
--> Asa class jyat pure virtual function ahe.
--> A class which contains at least one pure virtual function.
--> Asa class jyacha object bnvu skt nhi, function ch nhi ahe tr object ksa bnvnar.
--> The class which has pure virtual function is called as Abstract class.
--> if class is a abstract class, then you can not instantiate that class i.e., we can not create object of that class.


PURE VIRTUAL FUNCTION
----------------------
--> Asa function jyacha code lihu skt nhi which is equited to 0.
--> The function which does not have code means which is equated to zero is called as Pure Virtual function.


Ex :
-----
	class Shape							// abstract class
	{
		virtual void draw() = 0;		// pure virtual function
	}
